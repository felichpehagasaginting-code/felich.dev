## Why Next.js App Router? (And Why SPAs are Dead)

<FadeIn>
I've seen far too many portfolios built as pure Single Page Applications (SPAs) using standard Vite or Create React App. Here is the brutal truth: for a content-heavy, SEO-reliant site like a portfolio, pure SPAs are a terrible choice.

By utilizing the **Next.js 14 App Router**, I get Server-Side Rendering (SSR) and Static Site Generation (SSG) out of the box. This means my site loads instantly, crawlers can actually read my content, and I don't force the user's device to download megabytes of JavaScript just to render a header. 
</FadeIn>

<HighlightBox type="info">
The performance difference is night and day. With Next.js, the First Contentful Paint (FCP) is near zero. Add in the dynamic **Progressive Web App (PWA)** integration, and this site installs directly on your mobile device as a native app—something a basic SPA simply cannot match.
</HighlightBox>

## The "SimpleIcons" CDN Disaster & Inline SVGs

<FadeIn>
Initially, I used a CDN (`simpleicons.org`) to load all the technology logos on my "Uses" and "Skills" pages. It seemed smart. It wasn't.

Relying on external CDNs for core UI assets introduces latency, relies on a third-party DNS resolving quickly, and totally breaks offline PWA capabilities. Every time the page loaded, the browser had to make 60+ individual HTTP requests to fetch those icons. **Unacceptable.**
</FadeIn>

<FadeIn>
**The Fix:** I migrated every single logo into a highly optimized, local inline SVG library (`SkillIcons.tsx`). 
- **Zero external network requests.**
- **Perfect offline support.**
- **Instant rendering.**
If you care about performance, stop relying on external CDNs for your vector graphics.
</FadeIn>

## The AI Chatbot: Why Gemini 2.0 Flash over OpenAI?

<FadeIn>
If you look at the bottom-left corner, you'll see **Felich AI**—a highly intelligent assistant built directly into this site. It doesn't just read static text; it can parse Markdown, render code blocks, and even **speak to you using Text-to-Speech (TTS)**.
</FadeIn>

<FadeIn>
Many would ask: *"Why not use GPT-4?"*

Because **Gemini 2.0 Flash** is undeniably superior for this specific use case. It provides incredibly fast streaming responses with virtually zero latency. When a user asks a question, they want an answer *now*, not in 5 seconds. Furthermore, the integration with Google's ecosystem and the superior context window deduplication ensures the bot never hallucinates or loses track of its role. It's fast, it's smart, and it's built to impress.
</FadeIn>
