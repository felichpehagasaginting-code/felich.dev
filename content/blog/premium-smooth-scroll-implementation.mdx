## Why Smooth Scrolling Matters?

In modern web design, especially for premium portfolios, the "feel" of a website is just as important as its visual aesthetics. Standard browser scrolling can often feel mechanical and rigid. To solve this, I've integrated **Lenis Smooth Scroll** into my portfolio to provide a luxurious, inertial scrolling experience.

## The Technical Implementation

The challenge with smooth scrolling is ensuring it doesn't clash with other animations or cause performance "jitter." Here's how I optimized the implementation:

### 1. Singleton Pattern for Stability
I used a React `useRef` to ensure that Lenis is initialized as a singleton. This prevents multiple instances from running simultaneously during React re-renders, which is a common cause of scrolling glitches.

```tsx
const lenisRef = useRef<Lenis | null>(null);

useEffect(() => {
  if (!lenisRef.current) {
    lenisRef.current = new Lenis({
      lerp: 0.05,
      wheelMultiplier: 1,
      syncTouch: true,
    });
  }
  // ...
}, []);
```

### 2. GSAP Ticker Integration
To eliminate jitter, I bypassed the standard `requestAnimationFrame` loop and synced Lenis directly with the **GSAP Ticker**. This ensures that all scroll-linked animations and the scroll position itself are updated in a single, perfectly timed loop matching the monitor's refresh rate.

### 3. Hard-Disabling Native Behavior
I used CSS with `!important` flags to force-disable the browser's native `scroll-behavior`. This prevents the browser's engine from trying to "help" with smooth scrolling, which often creates conflicting movements.

```css
html, body {
  scroll-behavior: auto !important;
}
```

## The Result

The result is a "floaty" yet weighted feeling. When you flick the scroll wheel, the page glides smoothly and decelerates gracefully. This subtle change significantly elevates the overall user experience, making the portfolio feel like a high-end digital product.
